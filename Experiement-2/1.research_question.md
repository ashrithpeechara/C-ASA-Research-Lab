# Experiment 02

# Title

Static Routing and Basic Network Connectivity using Linux Router

---

# Objective

To understand Layer-3 routing by creating a custom Mininet topology consisting of two different IP networks connected through a Linux router. The experiment demonstrates IP forwarding, static routing, default gateways, and packet forwarding between different subnets.

---

# Problem Statement

In Experiment 01, communication occurred between hosts located within the same subnet using Layer-2 switching. However, communication between different IP networks requires a router.

Design and implement a custom Mininet topology that consists of:

- Two Hosts
- Two Open vSwitches
- One Linux Router

Configure the router to perform packet forwarding between two different subnets.

---

# Tasks

1. Create a custom Mininet topology using Python.
2. Build two independent LANs.
3. Configure a Linux router.
4. Enable IPv4 forwarding.
5. Assign IP addresses.
6. Configure default gateways.
7. Verify routing tables.
8. Test end-to-end connectivity.
9. Capture and analyze ICMP packets.
10. Understand Layer-3 packet forwarding.

---

# Expected Learning Outcomes

After completing this experiment, students should be able to:

- Understand Layer-3 routing
- Configure Linux as a router
- Enable IP forwarding
- Configure default gateways
- Understand static routing
- Analyze routing tables
- Capture routed packets using tcpdump
- Differentiate Layer-2 switching and Layer-3 routing

---

# Software Required

- Ubuntu Server
- Python3
- Mininet
- Open vSwitch
- tcpdump
- traceroute
