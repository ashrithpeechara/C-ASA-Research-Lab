# Experiment 03: Dynamic Routing using FRRouting (OSPF)

## Research Question

How can dynamic routing using the Open Shortest Path First (OSPF) protocol, implemented through FRRouting within Docker-based virtual routers, improve network scalability, adaptability, and routing efficiency compared to traditional static routing?

---

## Background

Traditional static routing requires network administrators to manually configure routing tables on every router. As the size and complexity of a network increase, static routing becomes difficult to manage, error-prone, and incapable of automatically adapting to topology changes.

Dynamic routing protocols such as OSPF enable routers to automatically discover neighboring routers, exchange routing information, calculate optimal paths, and update routing tables without manual intervention.

This experiment investigates the implementation of OSPF using FRRouting (FRR) deployed inside Docker containers to simulate an enterprise routing environment.

---

## Objectives

- Deploy virtual routers using Docker.
- Configure FRRouting on each router.
- Enable OSPF dynamic routing.
- Establish OSPF neighbor relationships.
- Verify automatic route learning.
- Compare dynamic routing with static routing implemented in Experiment 02.

---

## Research Goals

The experiment aims to evaluate:

- Automatic route discovery
- Network scalability
- Adaptability to topology changes
- Ease of network management
- Foundation for autonomous networking research

---

## Expected Outcome

By implementing OSPF, routers should automatically exchange routing information, maintain updated routing tables, and eliminate the need for manual route configuration. This dynamic routing infrastructure serves as the networking foundation for subsequent autonomic networking and Cognitive ASA experiments.
