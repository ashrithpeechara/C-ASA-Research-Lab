# Experiment 04 – Traffic Monitoring

## Research Question

How can a Docker-based autonomous network continuously monitor router health,
routing status, and traffic flow while ensuring reliable communication between
routers using automated monitoring techniques?

## Problem Statement

Modern autonomous networks require continuous monitoring of routing
protocols, interface statistics, and network health. Manual verification is
time-consuming and does not scale well. An automated monitoring mechanism
is required to provide real-time visibility into the network.

## Objectives

- Verify router availability.
- Monitor FRRouting services.
- Verify OSPF neighbor relationships.
- Validate routing tables.
- Monitor interface traffic statistics.
- Generate and observe network traffic.
- Automate health verification using Bash scripts.

## Expected Outcome

A monitoring script should successfully identify the operational state of
each router, verify OSPF connectivity, display routing information, and
report interface statistics without requiring manual inspection.
