# Experiment 04 Solution

# Title

Traffic Monitoring and Network Health Verification using Docker-based FRRouting Routers

---

# Objective

The objective of this experiment is to monitor the health of a Docker-based routing infrastructure by verifying router status, OSPF neighbor relationships, routing tables, interface statistics, and end-to-end connectivity using an automated Bash monitoring script.

---

# Lab Environment

| Component | Version |
|------------|----------|
| Ubuntu Server | 24.04 LTS |
| Docker Engine | Latest |
| Docker Compose | Latest |
| FRRouting (FRR) | Latest |
| Bash | GNU Bash |

---

# Theory

Continuous network monitoring is an essential component of modern network management. It enables administrators to observe router availability, routing protocol status, interface utilization, and network connectivity in real time.

In this experiment, a custom Bash script is used to automate health monitoring of a three-router Docker topology running the OSPF routing protocol. The script collects operational information from each router, verifies routing stability, and displays interface traffic statistics without requiring manual inspection.

# Procedure

## Step 1

Start the Docker-based routing infrastructure.

```bash
docker compose up -d
```

---

## Step 2

Verify all router containers are running.

```bash
docker ps
```

---

## Docker Containers Screenshot

![Containers](./screenshots/02-docker-ps.png)

---

## Step 3

Generate network traffic between Router1 and Router3.

```bash
docker exec router1 ping -c 20 10.0.23.3
```

Result

```
20 packets transmitted
20 packets received
0% packet loss
```

## Connectivity Test Screenshot

![Ping Test](./screenshots/03-ping-test.png)

---

## Step 4

Execute the automated health monitoring script.

```bash
./health_check.sh
```

The script verifies:

- Docker container status
- Router availability
- IP addresses
- FRRouting processes
- OSPF neighbors
- Routing tables
- Interface traffic statistics

# Observations

- All Docker router containers were operational.
- FRRouting services were running successfully on all routers.
- OSPF neighbors reached the FULL state.
- Remote networks were successfully learned through OSPF.
- End-to-end connectivity was verified using ICMP.
- No packet loss was observed during traffic generation.
- Interface statistics showed increased RX and TX packet counters after traffic generation.
- No interface errors or dropped packets were detected.

Reliable traffic monitoring is an important capability of autonomous networking systems. Automated health verification enables continuous observation of routing behavior, interface performance, and network connectivity without manual intervention.
